File Information Collection Policy
==================================
1.A QStringList will contains all the files waiting to be filtered.
2.KNMediaSeacher will receive the QStringList, it will check the suffix of the file, if it is in the list of the suffix, KNMediaSeacher will send a signal to KNMediaPlugin, and the plugin will tell the main model to add this file.
3.The model get a file path, and then add to the model, and will display the file name. The model will ask the KNMediaInfoCollectorManager to update the media file.
4.KNMediaInfoCollectorManager will update the media info.

KNMusicPlugin
|-KNMusicSeacher(Knows KNMusicModel)
|-KNMusicModel(Knows KNMusicInfoCollectorManager)
|-KNMusicInfoCollectorManager
Nerve Media Database File Format
================================
Overview:
1.All the text will be stored as UTF-8.
2.The image will be stored outside of the database, will use the row() of the item as the file name. E.g: the 5th row’s image will be store as: 5.png.
